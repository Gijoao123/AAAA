# Sei la

Issues Tracking é um negocio usado para o criador do repositorio do github ver os problemas do codigo e ver oque que tem pra mudar, os usuarios podem criar um tipo de negocio pra escrever o erro que tem no codigo ou no jogo, ai o criador do repositorio podem ver os erros e corrigir mais facilmente do que testar pessoalmente

Pull Requests ẽ um negocio que e feito para falar oque ele mudaram no codigo ou no jogo, para que os usuarios que usam o repositorio podem ver oque foi corrigido ou oque foi adicionado no repositorio
